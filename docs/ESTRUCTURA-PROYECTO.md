# Estructura del proyecto

## `index.html`
Archivo principal de la landing page. Contiene la estructura semántica, las secciones y el sprite SVG de íconos.

## `assets/css/styles.css`
Hoja de estilos principal. Aquí se deben hacer los cambios visuales: colores, tamaños, responsive, tarjetas, botones, header, footer y formularios.

## `assets/js/main.js`
Contiene la lógica del menú móvil, cierre automático de navegación, año dinámico del footer, formulario simulado y ajustes de ancho para integración tipo Elementor.

## `assets/img/`
Carpeta preparada para imágenes locales. Actualmente se deja un README porque el código original usa imágenes externas de WordPress.

## `docs/`
Documentación de apoyo para mantenimiento del proyecto.

## `original/`
Copia intacta del archivo recibido. Úsala como respaldo si necesitas comparar cambios.

## Recomendación para siguiente fase

Cuando confirmes que la página abre bien en Antigravity, el siguiente paso lógico es convertir las imágenes remotas a archivos locales y luego conectar el formulario a WhatsApp, correo, Formspree o un backend propio.
